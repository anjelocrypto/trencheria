## What I'm building

**Trencheria** is a **3D multiplayer browser game** with:
- 7 factions/clans competing in a shared online world
- Territory wars between clans
- Player progression and in-game coins
- **Phantom wallet (Solana)** authentication — players sign in with their crypto wallet
- Real-time multiplayer presence and chat
- Medieval/fantasy theming

The game runs entirely in the browser using **React Three Fiber** (Three.js) for the 3D world.

## Technical stack

### Monorepo structure (pnpm workspaces)
```
/
├── artifacts/
│   ├── trencheria/          ← main game frontend (the actual game)
│   ├── api-server/          ← Express API scaffold (not used by game yet)
│   └── mockup-sandbox/      ← design preview sandbox
├── lib/
│   ├── api-spec/            ← OpenAPI spec + generated client
│   ├── api-client-react/    ← React Query hooks generated by Orval
│   ├── api-zod/             ← Zod schemas for validation
│   └── db/                  ← Drizzle schema (scaffold only)
├── scripts/                 ← build/setup scripts
├── .migration-backup/       ← original Lovable code (read-only reference)
└── package.json             ← workspace root
```

### Frontend (`artifacts/trencheria`)
- **React 19** + **Vite 7** + **TypeScript 5.9**
- **Tailwind CSS v3** (PostCSS-based) + **shadcn/ui** components (Radix UI primitives)
- **React Three Fiber v9** + `@react-three/drei` v10 for 3D
- **react-router-dom v6** for routing
- **@tanstack/react-query** for server state
- **react-hook-form** + **zod** for forms

### Backend
- **Supabase** is the primary backend — kept exactly as it was on Lovable
  - 30+ **RPC calls** (Postgres stored procedures for clans, territories, wars, coins, leaderboard, chat)
  - **Realtime channels** for multiplayer (presence + broadcast)
  - **Edge Functions** — notably `verify-wallet` for Phantom signature verification
- **Express 5 API server** in `artifacts/api-server` exists as a scaffold but the game does NOT use it yet
- **Drizzle ORM** scaffolding exists but actual game data lives in Supabase

### Auth
- **Phantom wallet (Solana)** — players sign a message with their wallet, the signature is verified by a Supabase Edge Function (`verify-wallet`), and they get a Supabase session

### Tooling
- `pnpm` for package management
- `Orval` for OpenAPI → React Query hook codegen
- `esbuild` for bundling the API server
- `tsc` for typechecking
