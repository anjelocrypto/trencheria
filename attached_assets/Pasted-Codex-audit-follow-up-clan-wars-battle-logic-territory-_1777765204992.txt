3. Replace admin-only winner flow with automatic territory result
Right now the UI says “Admin resolves winner.” That is not good enough for gameplay.

Design and implement automatic resolution:
- War starts after pending countdown.
- War lasts the configured battle duration.
- Kills during the active war count toward attacker/defender score.
- When the war ends:
  - attacker wins if attacker_kills > defender_kills
  - defender holds if defender_kills >= attacker_kills
  - territory owner updates automatically on attacker win
  - territory enters cooldown
  - territory_history records the result
  - challenge status becomes completed/resolved
- Keep AdminWorldMap manual resolve as an override, not the main gameplay path.

If this requires DB function changes, create a new `supabase/pending/...sql` file only. Do not apply.
